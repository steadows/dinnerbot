# Project Blueprint: "DinnerBot" - Weekly Meal Planner & Grocery Assistant

## 1. Project Overview

**Goal:** Create a serverless application that runs weekly to generate dinner options using an LLM (Gemini), sends them via Telegram to the family, waits for a selection (via inline keyboard buttons), and replies with a consolidated grocery list.

**Infrastructure:** Google Cloud Platform (GCP)
**IDE/Assistant:** Cursor (with GSD framework)
**Language:** Python 3.10+
**Messaging (Primary):** Telegram Bot API
**Messaging (Optional Extension):** Twilio SMS (pending phone number verification)

---

## 2. Architecture & Tech Stack

### High-Level Flow
1.  **Trigger:** Google Cloud Scheduler (Cron job: every Sunday at 2 PM).
2.  **Generator:** Google Cloud Function A (`generate_options`) calls Gemini API to create recipes.
3.  **Storage:** Google Firestore stores the generated options with a unique `session_id`.
4.  **Outbound:** Telegram Bot API sends the options as a formatted message with inline keyboard buttons to the family chat.
5.  **Inbound (Reply):** User taps an inline button or sends a text reply. Telegram Webhook triggers Cloud Function B (`handle_update`).
6.  **Processor:** `handle_update` retrieves the session from Firestore, asks Gemini to convert the selection into a grocery list, and sends the list back via Telegram.

### Tech Stack
* **Compute:** Google Cloud Functions (2nd Gen)
* **Database:** Google Firestore (NoSQL, Serverless)
* **Scheduling:** Google Cloud Scheduler
* **LLM:** Google Gemini API (via `google-generativeai` lib)
* **Messaging (Primary):** Telegram Bot API (via `python-telegram-bot` lib)
* **Messaging (Optional):** Twilio SMS (see Extension A)

## 4. Security Considerations

### Telegram Webhook Validation
Telegram supports a `secret_token` parameter when setting the webhook. Every incoming update will include an `X-Telegram-Bot-Api-Secret-Token` header that must match the configured secret. This prevents spoofed webhook requests.

```python
SECRET_TOKEN = config.get_value("TELEGRAM_WEBHOOK_SECRET", is_secret=True)

def validate_telegram_request(request):
    token = request.headers.get("X-Telegram-Bot-Api-Secret-Token")
    if token != SECRET_TOKEN:
        return False
    return True
```

### Secrets Management
* **Local Development:** Use `.env` file (never commit to git).
* **Production (Cloud Functions):** Use **Google Secret Manager** to store:
  - `GEMINI_API_KEY`
  - `TELEGRAM_BOT_TOKEN`
  - `TELEGRAM_WEBHOOK_SECRET`
  - (Extension A) `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`

### Allowed Chat IDs
Restrict the bot to only respond to authorized chat IDs to prevent strangers from using it:
```python
ALLOWED_CHAT_IDS = [123456789]  # Your family chat ID(s)
```

## 7. Error Handling Strategy

### LLM Response Handling
* **JSON Parsing:** Wrap Gemini responses in try/except. If JSON is malformed, retry up to 2 times with a more explicit prompt ("Return ONLY valid JSON, no markdown").
* **Empty/Invalid Response:** If Gemini returns empty or unusable content after retries, use the fallback recipe set defined in `llm_service.py`.

### Telegram Failures
* **Send Failure:** Log the error and store the message in Firestore with `status: send_failed` for manual review or retry.
* **Webhook Errors:** Always return HTTP 200 to Telegram, even on internal errors. Telegram will retry on non-200 responses, which can cause duplicate processing.
* **Rate Limits:** Telegram allows ~30 messages/second to different chats (not an issue for a family bot), but implement exponential backoff if 429 errors occur.

### Twilio Failures (Extension A)
* **Send Failure:** Log the error and store with `status: send_failed`.
* **Webhook Errors:** Always return valid TwiML, even on errors, to prevent Twilio from retrying indefinitely.

### User-Facing Error Messages
Keep error messages friendly, in-character (Gordon Ramsay), and actionable:
* No active session: "No meal options waiting for you right now, love. Check back on Sunday!"
* Invalid selection: "I didn't catch that. Tap one of the buttons or reply 1, 2, or 3 to pick your dinner."
* System error: "Bloody hell, something went wrong on my end. Give it another go in a few minutes."
